What is HTML?

  • Hyper Text Markup Language

    Markup language for web pages. I like to think of it as a data format for web browsers.

                
<!DOCTYPE html>
<html>
    <head>
        <title>Hello World</title>
    </head>
    <body>
        <p>Hello World!</p>
    </body>

<div> and <span>

Used When No Other Semantic Element Will Work

The <div> and <span> tags are used when you have no other semantic element that works. For example, although it is possible to place a list or a paragraph inside a <div> while maintaining the same rendering, it is not a very good idea because we have the more semantic elements <ul> and <p>.

<div> and <span>

Inline vs Block

Inline elements do not provide a box for the content that is contained within them, and will not provide a line break. Block elements do.

CSS

Cascading Style Sheets is a language to describe the look and format of the document. Elements in the document are targetted using selectors and provided rules to enact on that selector.

Syntax is easy to learn.

        
        p {
            color: #000; 
        }

        #main {
            width: 968px;
        }

        .widget {
            margin-left: 10px;
            padding: 5px;
        }

CSS

Syntax and Selectors

  • Syntax

    selector { rules; }

  • Element Selector

    p { color: #000; }

  • ID Based Selector

    #main { color: #000; }

  • Class Based Selector

    .widget { color: #000; }

ID vs. Class

ID's and classes are attributes that are set on elements.

ID

Used for elements that appear on the page once.


    <div id="main">
        ...
    </div>

    #main {
        width: 968px;
    }

ID vs. Class

Class

Used for element groups that you will reuse.


    <div class="widget">
        ...
    </div>

    .widget {
        background: url(somewhere.png);
        height: 300px;
        width: 300px;
    }

Summary

Use id's when you will only have one possible element to target. Use classes when you will have rules that you need to reuse in multiple places on the page.

Grouping Selectors

What happens when you need to apply the same rule to multiple selectors?

  • Use Selector Grouping

    h1 { color: #fff; }
    h2 { color: #fff; }
    h3 { color: #fff; }

    This is the same as...

  • h1, h2, h3 { color: #fff; }

  • Grouping Classes or ID's

    .widget, .module, .plugin { color: #fff; }

    #main, #person, #photo { width: 500px; }

Selecting a Element With a Specific Id or Class

h1.widget { color: #fff; }

h1#main { color: #fff; }

Use the . or # syntax to denote that an element has a particular id or class.

Parents, Descendants, and Children

  • .widget p { padding: 5px; }

    The rule will apply to any p descendant of a class widget.

  • .widget > p { padding: 5px; }

    The rule will apply to any p child of a class widget.

  • .widget ul > li { padding: 5px; }

    The rule applies to the list element child of an unordered list that is a descendant of a class widget.

  • Display

    All elements have a default display. For example, p is a block level element, and span is an inline element.

    With css you can alter the default to display.

    <p>This is a paragraph with <span>a span inside</span> and some more text.</p>

  • Changing the default

    <p>This is a paragraph with <span>a span inside</span> and some more text.</p>

    span { display: block; }

  • We get...

    This is a paragraph with a span inside and some more text.

JavaScript - The Language of the Web

  • Quick Intro

    Javascript is a prototype based multi paradigm language.

  • Objects are cloned rather than created by classes

  • Object oriented with some functional goodness

  • Functions are first class objects

    Meaning we can pass them in as parameters to other functions.

  • Interacts with the web browser through an API called the DOM (Document Object Model)

JavaScript

  • JavaScript !== Java

    The two languages have about as much in common as peanut butter and ham.

  • JavaScript is more like Lisp or Scheme than Java

    It's like a Scheme with a C style syntax.
